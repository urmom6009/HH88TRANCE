# Privacy

The service collects only what is needed to fulfill digital purchases:

- Telegram ID
- Telegram username and first name when Telegram provides them
- Stripe customer/payment references when needed for support
- Product purchase history
- Delivery/access status
- Minimal audit metadata

The service does not store card numbers, full billing details, unnecessary addresses, or unrelated user messages.

Delivery-token logs should avoid full IP addresses and full user agents unless a specific abuse investigation requires them. Backups contain user/payment metadata and must be protected.

Manual deletion or anonymization should remove Telegram profile fields where feasible while preserving payment/accounting records required for fraud, refund, or legal handling.
